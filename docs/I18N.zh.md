# 国际化（i18n）指南

NetTools Hub 面向全球用户构建。本文档说明翻译如何运作，以及如何添加或改进翻译。

> 🇨🇳 简体中文 · 🇬🇧 [English](./I18N.md) · 🇯🇵 [日本語](./I18N.ja.md)

---

## 1. 代码中的 i18n 怎么运作

### 1.1 一份文件，三种语言

所有 UI 文案集中在 **一份文件**：[`src/lib/i18n.ts`](../src/lib/i18n.ts)。它是一张三列扁平键值表，约 36 个键：

```ts
const translations = {
  en: {
    "site.title": "NetTools Hub",
    "nav.home": "Home",
    "taxonomy.kind": "Kind",
    "table.col.name": "Name",
    // …
  },
  zh: {
    "site.title": "NetTools Hub",
    "nav.home": "首页",
    "taxonomy.kind": "类别",
    "table.col.name": "名称",
    // …
  },
  ja: {
    "site.title": "NetTools Hub",
    "nav.home": "ホーム",
    "taxonomy.kind": "種別",
    "table.col.name": "名称",
    // …
  },
};
```

另外 [`src/lib/taxonomy.ts`](../src/lib/taxonomy.ts) 里有一张
**第二张小表**，用于 kind 和 platform 的人类可读标签（如
`proxy` → "Proxy cores" / "代理核心" / "プロキシコア"）。把它
拆出来是为了让标签跟着分类走 —— 加一个新 `kind` 只需要改
一个文件。

当前语言在运行时从 `?lang=` URL 参数读取，并以
`localStorage` 作为粘性偏好、`navigator.language` 作为最终
回退。**不使用 cookies。** 深链是稳定 URL —— 分享
`?lang=zh` 和分享 `?lang=ja` 一样好用。

### 1.2 `t()` 函数

```ts
import { t } from "@/lib/i18n";

t("en", "site.title");                          // "NetTools Hub"
t("zh", "nav.home");                            // "首页"
t("ja", "taxonomy.kind");                       // "種別"

t("en", "taxonomy.count", { n: 12 });           // "12 entries"
t("zh", "taxonomy.count", { n: 12 });           // "12 条"
t("ja", "taxonomy.count", { n: 12 });           // "12 件"
```

占位符语法为 **`{name}`**（大括号两侧无空格）。

### 1.3 缺失的键

如果某种语言缺某个键，`t()` 会回退到英文，并在 console 打印警告 —— 让站点仍能渲染，同时提醒我们补齐。

## 2. 添加新语言

> 想添加 西班牙语 / 法语 / 韩语 / 俄语 / 阿拉伯语 / ……？非常欢迎！方法如下。

### 步骤 1 — 打开 i18n 文件

```bash
$EDITOR src/lib/i18n.ts
```

### 步骤 2 — 注册语言

在 [`Lang`](../src/lib/i18n.ts) 联合类型中加你的代码：

```ts
export type Lang = "en" | "zh" | "ja" | "es";   // ← 加 "es"
```

并在同一文件中的 `LANG_OPTIONS`（语言切换器用到的数组）里加：

```ts
export const LANG_OPTIONS = [
  { value: "en", label: "EN" },
  { value: "zh", label: "中文" },
  { value: "ja", label: "日本語" },
  { value: "es", label: "Español" },   // ← 加
] as const;
```

### 步骤 3 — 在翻译表中加一列

用 **与 `en` 完全相同的键**新增一项：

```ts
const translations = {
  en: { … },
  zh: { … },
  ja: { … },
  es: {                                          // ← 新
    "site.title": "NetTools Hub",
    "nav.home": "Inicio",
    // …
  },
};
```

> 💡 最快的方式是复制 `en` 整段并逐个翻译。缺失的键会回退英文，所以也可以先提个草稿 PR 慢慢补全。

### 步骤 4 — 更新 `isValidLang()` 与 `getLangFromUrl()`

在 `src/lib/i18n.ts` 找到这两个 helper：

```ts
export function isValidLang(v: string | null | undefined): v is Lang {
  return v === "en" || v === "zh" || v === "ja" || v === "es";   // ← 更新
}
```

### 步骤 5 — 添加对应语言的 README

```bash
cp README.md README.es.md
# 翻译正文
```

GitHub 会自动识别 `README.<lang>.md` 文件。

### 步骤 6 — 在 `LANG_OPTIONS` 加切换器项

第 2 步已经做了。

### 步骤 7 — 提 PR

在 PR 描述中说明：

- 添加了哪种语言
- 你是母语 / 流利使用者
- 是否也翻译了 README（推荐）

## 3. 改进现有翻译

发现错别字或别扭的措辞？两种方式：

### 方式 A — 直接改 i18n 文件

```bash
$EDITOR src/lib/i18n.ts
```

找到对应键（如 `"empty.description"`），修改值，commit，push，提 PR。

### 方式 B — 先开 Issue

如果拿不准措辞，开一个 [翻译 Issue](../../issues/new?template=translation.yml) 一起讨论。

## 4. 翻译风格指南

| 准则 | 示例 |
|---|---|
| **简短** —— UI 标签受宽度限制 | "Explore" / "探索" / "探索"（3 字） |
| 除非必要，**不翻译**技术术语 | `Clash`、`sing-box`、`WebSocket`、`CDN` 保持原样 |
| **匹配语气** —— 简洁、专业、友好 | 不要添加源语言没有的 emoji |
| **保留占位符** —— `{count}`、`{n}` 必须保留 | `t(lang, "taxonomy.count", { n })` |
| **LANG_OPTIONS 标签用对应语言** | "中文" 而不是 "Chinese" |

---

## 5. 验证翻译

```bash
pnpm dev     # http://localhost:8080/?lang=es
```

1. 打开每个页面（`/`、`/explore`）
2. 点击右上角 Globe 图标
3. 选择你的新语言
4. 检查每个标签、错误状态、404 页面、空状态、"Clear filters" 按钮等
5. 如果有缺失的字符串，会回退英文并在 console 警告 —— 补齐即可
